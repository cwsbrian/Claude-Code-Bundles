**Step 3: Create `packages/cli/src/setup.ts`:**

Export two functions:
- `installCommands(force: boolean = false): Promise<string[]>` -- copies .md files from package's `commands/bundle/` to `~/.claude/commands/bundle/`
- `runSetup(args: string[]): Promise<void>` -- CLI entrypoint, parses --force flag, calls installCommands, prints summary

Key implementation details for setup.ts:
- Source path: `path.resolve(path.dirname(fileURLToPath(import.meta.url)), "..", "commands", "bundle")` (resolves to package's commands/bundle/ relative to dist/)
- Dest path: `path.join(os.homedir(), ".claude", "commands", "bundle")`
- Import `cp, mkdir, readdir, stat` from `"node:fs/promises"`, `path` from `"node:path"`, `os` from `"node:os"`, `fileURLToPath` from `"node:url"`
- `installCommands` creates dest dir with `mkdir({ recursive: true })`, reads source dir with `readdir`, iterates .md files
- If `force=false` and dest file exists (`stat` does not throw), skip with message "Skipped {dest} (already exists, use --force to overwrite)"
- If `force=true` or file doesn't exist, `cp(src, dest)` and add to installed list
- Returns array of installed file paths
- `runSetup` checks `args.includes("--force")`, calls `installCommands(force)`, prints summary with count and usage hint

**import.ts changes (D-04: auto-install command files as side-effect of import):**

1. Add import at top of file:
```typescript
import { installCommands } from "./setup.js";
```

2. After the final `process.stdout.write(`Imported ${owner}/${slug} -> ${applyResult.claudeRoot}\n`);` line (the very last line of runImport), add:
```typescript
  // D-04: Auto-install command files as side-effect of import
  try {
    const installed = await installCommands(false);
    if (installed.length > 0) {
      process.stdout.write(`\nAlso installed ${installed.length} Claude Code command file(s).\n`);
      process.stdout.write("Use /bundle:import, /bundle:pull, /bundle:status, /bundle:browse in Claude Code.\n");
    }
  } catch {
    // Non-fatal: command file installation failure should not break import
  }
```

The `installCommands(false)` call with `force=false` means it skips existing files, so repeated imports don't overwrite custom modifications. The try/catch ensures installation failure is non-fatal.
